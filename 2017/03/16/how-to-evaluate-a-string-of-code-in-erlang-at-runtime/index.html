与Erlang有关的安全问题是什么？！

相似地，我们可能会碰到表达式代码的安全问题。

我们被允许包含任何函数 – 本地函数、BIF（Erlang的内建函数）和在你创建的其他模块里暴露出来的函数 – 的调用，并且它将解析它们然后尝试执行它们。

如果我们让上述函数能被外界访问，即使是间接的，而且输入没有经过净化，那么我们就是给别人直接调用各种他们没有业务调用函数的能力。危险啊！

那么我们如何防止这样的危险呢？
